1. Знакомим git с собой
git config --global user.name "name"
git config --global user.email "adres"

2. Создание репозитория
В GitHub создаём репозиторий (папка пример ProjectTest)
Копипастим ссылку
На рабочем столе креэйтим папку

В  консоле git:
3. git clone https:// наша ссылка ENTER
Клонировали Долли

4.  cd <путь_к_проекту>

5. #Инициализация/создание репозитория
git init
Позволяет из папки сделать репозиторий (папка, в которой настроена система контроля версий)
То, с чего git начинает работать с папкой.


5. Добавить файлы отслеживанию программы (add - добавь):
все - git add .
или отдельный файл - git add "name file"

Теперь Git отслеживает изменения файлов проекта. 
Но, так как только создали репозиторий в нем нет кода. 
Для этого необходимо создать commit.

6. Делаем commit
Сохраняет текущее состояние файла + текстовый комментарий обновления
git commit -m "комментарий того, а шо мы, собственно, сделали"

____Аллилуйя. Мы сделали.

Так же комманды:

7. git diff
отличие текущего файла от сохраненного (закоммиченного)

8. git log
Журнал всех изменений

9. Создание новой ветки/список веток
git branch название_ветки

10. Переключение между ветками 
git checkout название_ветки 

11. Обновляем ветку с сервера
git checkout название_ветки
git pull origin название_ветки

11. Закинуть в гит хаб
git checkout название_ветки
git push origin название_ветки 

12. Слияние
git merge 
git checkout название_ветки (куда добавляем)
git merge название_ветки (которую добавляем)

13. Удаление ненужной ветки
 git branch -d название_ветки
